Chapter 2.0 输出流(Output Streams)

		java.io.OutputStream类定义了三个向流写字节数据的基本方法.它也有用来close和flush流
	的方法:
		public abstract void write(int b) throws IOException
		public void write(byte[] data) throws IOException
		public void write(byte[] data, int offset, int length) throws IOException
		public void flush( ) throws IOException
		public void close( ) throws IOException
		
		OutputStream是一个抽象类.子类提供write(int b)抽象方法的实现.他们也可以重写其他四个
	非抽象方法.比如,FileOutputStream类重写了所有五个方法以使用native代码写文件.尽管
	OutputStream 是抽象的,但是一般来讲你只需要知道你操作的对象是一个OutputStream;更多特定的
	OutputStream的子类都被藏起来了.比如java.net.URLConnection中的getOutputStream()方法签名是
	这样的:
		public OutputStream getOutputStream( ) throws IOException

		根据URLConnection对象关联的URL的不同,实际返回的输出流类型可能是
	sun.net.TelnetOutputStream,sun.net.smtp.SmtpPrintStream,sun.net.www.http.KeepAliveStream
	或者其他完全不同的类.但是对程序员来讲,你需要知道的只是它返回了一个OutputStream子类的对象.
	
		此外,即使你知道你所操作的子类是什么类型,你仍然需要使用从OutputStream中继承的方法.
	即使这些继承的方法没有在子类的API文档中出现,记住他们是从哪来的就会变得很重要.比如,在
	java.io.DataOutputStream中没有定义close()方法,但是你仍然可以使用从超类中继承来的这个方法.
